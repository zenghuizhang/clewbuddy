# 🦞 ClewBuddy — AI 语音桌面伴侣

## 产品概述

**ClewBuddy** 是一个基于 M5Stack CoreS3 的 AI 语音交互桌面伴侣设备。用户按住屏幕说话，设备将语音通过 WiFi 发送到本地中转服务器，由 AI 模型处理后返回文字+语音回复，设备播放语音并在屏幕上显示表情动画。

**核心卖点：把你的 AI 助手从屏幕里解放出来，变成一个能听能说、有表情的实体伙伴。**

## 为什么选这个项目

1. **半天可完成** — 架构简单：固件 + 中转服务器，两个模块
2. **展示效果好** — 现场演示时，对着设备说话、设备回答，比纯软件项目更有冲击力
3. **充分利用硬件** — 麦克风(语音输入) + 喇叭(语音输出) + 屏幕(表情) + WiFi(联网) + 触屏(交互)
4. **与 OpenClaw/AI Agent 强相关** — 天然契合黑客松主题

## 用户流程

```
用户按住屏幕 → 录音 → 松开 → 
设备显示"思考中"表情 → 
音频通过WiFi发送到本地Server → 
Server调用 Whisper(语音转文字) → 
Server调用 LLM(生成回复) → 
Server调用 TTS(文字转语音) → 
返回文字+音频给设备 → 
设备播放语音 + 显示"说话"表情 + 滚动显示文字
```

## 技术架构

```
┌─────────────────┐     HTTP/WiFi      ┌──────────────────────┐
│  M5Stack CoreS3 │ ◄────────────────► │   Local Python Server │
│                 │                     │                       │
│ • 录音 (16kHz)  │  POST /chat         │ • Whisper STT         │
│ • 播放 (PCM)    │  ──────────────►   │ • LLM API (通义千问)   │
│ • 屏幕表情      │                     │ • Edge TTS            │
│ • 触屏交互      │  WAV audio + text   │                       │
│                 │  ◄──────────────   │                       │
└─────────────────┘                     └──────────────────────┘
```

### 固件端 (C++ / PlatformIO)
- **M5Unified** 统一 API
- 录音：16kHz 16bit mono PCM，按住录制
- 网络：HTTP POST 发送 WAV 数据到服务器
- 播放：接收 PCM/WAV 音频数据并播放
- 显示：简单像素表情动画

### 服务器端 (Python)
- **FastAPI** 轻量 HTTP 服务
- **openai-whisper** 或 **faster-whisper** 语音识别
- **LLM**: 调用本地或云端模型（优先用已有的API）
- **edge-tts**: 微软免费TTS，中文效果好，零配置

## 表情设计

```
😊 待机 (Idle)        — 眨眼动画，偶尔晃动
🎤 录音 (Recording)   — 嘴巴张开，声波动画
🤔 思考 (Thinking)    — 眼睛转圈，"..."气泡
🗣️ 说话 (Speaking)    — 嘴巴开合动画
```

## 风险与备选

| 风险 | 备选方案 |
|------|----------|
| Whisper 本地太慢 | 用在线 STT API |
| 音频播放有杂音 | 降低采样率，加滤波 |
| WiFi 不稳定 | 重连机制 + 错误提示表情 |
| LLM API 不可用 | 预置几条固定回复 |
